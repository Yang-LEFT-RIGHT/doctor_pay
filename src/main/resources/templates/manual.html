第1章：系统概述

1.1 系统简介与定位

DocIM Stipend 博士生工作管理系统是一个专门为高校博士生设计的综合性工作管理平台。系统致力于解决博士生在承担助教、代管等工作中面临的任务管理、时间记录和统计报告等问题。

系统目标
  • 提供统一的工作任务管理平台
  • 实现工作时间的精准记录和统计
  • 简化工作报告的生成和提交流程
  • 提升博士生工作效率和管理水平
  • 支持多角色协同工作

系统架构图：

表现层
Web界面 移动端适配 响应式设计
应用层
任务管理 工时统计 报告生成 通知系统
业务逻辑层
权限控制 工作流程 数据验证
数据层
MySQL数据库 文件存储 缓存系统

1.2 系统架构与设计原则

系统采用分层架构设计，确保系统的可扩展性、可维护性和安全性。

可扩展性

模块化设计，支持功能扩展和定制开发

安全性

多层次安全防护，数据加密传输和存储

高性能

优化数据库查询，支持高并发访问

响应式

支持PC、平板和手机多种设备访问

技术栈：

层级 技术选型 说明
前端 HTML5, CSS3, JavaScript 响应式设计，支持现代浏览器
后端 Node.js, Express.js 高性能服务端框架

缓存 Redis 提升系统响应速度

第2章:入门指南

2.1 系统访问与环境要求

硬件要求

  • CPU：双核 2.0 GHz 或更高
  • 内存：4GB 或以上
  • 硬盘：至少 500MB 可用空间

软件要求

  • 操作系统：Windows 10+ / macOS 10.14+ / Linux
  • 浏览器：Chrome 90+ / Firefox 88+ / Safari 14+
  • 网络：稳定的互联网连接

插件要求

  • 启用 JavaScript
  • 启用 Cookies
  • PDF 阅读器（用于查看报告）

访问方式：

Web浏览器访问

推荐使用最新版本的 Chrome 或 Firefox 浏览器访问系统。

系统网址
https://docim.your-university.edu
移动端访问

系统支持移动端访问，可通过手机浏览器访问系统网址。

移动端建议使用横屏模式以获得更好的操作体验

访问故障排查：

问题：无法访问系统

解决方案：

  1. 检查网络连接是否正常
  2. 确认系统网址是否正确
  3. 清除浏览器缓存并重试
  4. 尝试使用其他浏览器

2.2 账号注册与激活

系统采用集中式账号管理模式，博士生账号由院系管理员统一创建和分配。

账号注册流程：

1
信息收集

院系管理员收集博士生基本信息，包括：

  • 学号
  • 姓名
  • 专业方向
  • 电子邮箱
  • 联系电话
2
账号创建

管理员在系统中创建账号，系统会自动生成：

  • 用户名（通常为学号）
  • 初始密码（随机生成）
  • 账号激活链接
3
账号分发

系统通过邮件发送账号信息，包含：

  • 系统访问地址
  • 用户名和初始密码
  • 账号激活说明
  • 技术支持联系方式
4
首次登录

博士生收到邮件后，按照指引完成：

  • 访问系统登录页面
  • 使用初始密码登录
  • 强制修改密码
  • 完善个人信息

重要注意事项

  • 初始密码为一次性密码，首次登录后必须修改
  • 账号激活邮件有效期为7天，请及时激活
  • 如未收到激活邮件，请检查垃圾邮件文件夹
  • 如忘记密码，可通过"忘记密码"功能重置

第3章:核心功能详解

3.2 任务管理系统

任务管理系统是平台的核心功能模块，支持任务的全生命周期管理。

任务生命周期：

创建
管理员/教师创建任务
发布
系统发布任务通知
接受
博士生接受任务
进行中
任务执行与记录
完成
提交任务报告
审核
管理员审核确认

任务详细信息：

字段 说明 示例
任务名称 任务的标题，应简洁明了 "《数据结构》课程助教"
任务类型 助教、代管、临时工作等 助教工作
需求人数 完成任务需要的人数 2人
预计工时 完成任务预计需要的时间 40小时
截止日期 任务必须完成的最后日期 2024-06-15
任务描述 详细的工作内容和要求 负责课程答疑、作业批改...
技术要求 完成任务需要的技能要求 熟悉Java编程、了解数据结构
奖励标准 完成任务的薪酬标准 40元/小时

操作指南：接受任务

1
查看待处理任务

登录系统后，在个人首页查看"待处理任务"列表

参考截图：个人首页待处理任务区域
2
查看任务详情

点击任务名称或"查看详情"按钮，进入任务详情页面

请仔细阅读任务描述、要求和截止日期
3
确认接受任务

在任务详情页面，点击"接受任务"按钮

注意：接受后不可随意取消，请谨慎操作
4
确认接受成功

系统会显示接受成功提示，任务状态更新为"已接受"

任务已成功接受，请按时完成任务
